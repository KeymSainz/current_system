====================================================================
  DOCUMENT APPROVAL SYSTEM - SETUP INSTRUCTIONS
====================================================================

STEP 1: Check if migration is needed
-------------------------------------
Open in browser:
http://localhost/current_system/fixandgo/backend/check_migration.php

If it says "document_approvals table DOES NOT EXIST", proceed to Step 2.
If it says "EXISTS", skip to Step 3.


STEP 2: Run the database migration
-----------------------------------
Option A - Via phpMyAdmin:
1. Open phpMyAdmin
2. Select your database (fixandgo)
3. Click "Import" tab
4. Choose file: fixandgo/backend/migrate_document_approvals.sql
5. Click "Go"

Option B - Via MySQL command line:
mysql -u root -p fixandgo < fixandgo/backend/migrate_document_approvals.sql


STEP 3: Test the system
------------------------
1. Log in as admin
2. Go to "Applicants" section
3. You should see 3 buttons for each applicant:
   - Docs (blue) - View documents
   - Approve (green, DISABLED initially)
   - Reject (red)

4. Click "Docs" button
5. You should see:
   - Each document with status badge (Pending/Approved/Rejected)
   - "Approve" and "Reject" buttons under each document
   - If a document was rejected, you'll see the rejection reason

6. Click "Approve" on a document
   - Document status changes to "Approved"
   - Green checkmark appears

7. Click "Reject" on a document
   - Prompt asks for rejection reason
   - Enter reason and confirm
   - Applicant gets notified via email and in-app notification

8. Once ALL required documents are approved:
   - Green banner appears: "All Documents Approved!"
   - "Approve Application" button appears in the modal
   - Main "Approve" button in the table becomes ENABLED

9. Click "Approve Application" to fully activate the seller account


TROUBLESHOOTING
---------------
If buttons don't appear:
- Clear browser cache (Ctrl+Shift+Delete)
- Hard refresh (Ctrl+F5)
- Check browser console for errors (F12)

If migration fails:
- Make sure you're using the correct database name
- Check if tables already exist
- Verify MySQL user has CREATE TABLE permissions
